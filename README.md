### About Me

I build and run the data systems that products quietly depend on. My focus is Snowflake development and database administration — designing warehouses, writing performant SQL, and keeping PostgreSQL, MySQL, and cloud databases fast, available, and recoverable. I currently work at Infosys and hold a graduate education from the University of Central Missouri.

I care about the unglamorous parts of data engineering: query plans that actually scale, backups that actually restore, and pipelines that don't page you at 3 a.m. When something is slow, I'd rather find out why than throw hardware at it.

---

### What I Do

- **Snowflake development** — warehouses, roles, tasks, streams, and SQL tuned for cost and speed.
- **Database administration** — PostgreSQL and MySQL setup, tuning, and day-to-day reliability.
- **Cloud data infrastructure** — AWS RDS and Aurora provisioning, scaling, and maintenance.
- **High availability & disaster recovery** — replication, failover, and backup/restore strategies.
- **CI/CD for data** — automated database changes and deployments through Azure DevOps.
- **Performance tuning** — indexing, query optimization, and diagnosing slow workloads.
